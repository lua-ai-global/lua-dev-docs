## Overview

The `lua logs` command provides an interactive interface for viewing and navigating agent and skill logs with real-time pagination.

```bash
lua logs
```

  <Card title="View All Logs" icon="list">

    Navigate, refresh, jump to page
  </Card>
</CardGroup>

## Log Types

<Tabs>
  <Tab title="❌ Error">
    **Execution errors and failures**
    
    - Tool execution failures
    - API errors
    - Invalid configurations
    - Stack traces for debugging
    
    Color: Red
  </Tab>

  <Tab title="⚠️ Warn">
    **Warnings and potential issues**
    
    - Performance warnings
    - Deprecated features
    - Configuration warnings
    
    Color: Yellow
  </Tab>

  <Tab title="🔍 Debug">
    **Debug information**
    
    - Tool inputs/outputs
    - Function execution status
    - Detailed operation info
    
    Color: Blue
  </Tab>

  <Tab title="ℹ️ Info">
    **General information**
    
    - Operation status
    - Configuration changes
    - General messages
    
    Color: Cyan
  </Tab>

  <Tab title="▶️ Start/✅ Complete">
    **Operation metrics**
    
    - Operation started
    - Operation completed
    - Duration measurements
    
    Color: Green
  </Tab>
</Tabs>

## Use Cases

### Debug Tool Errors

```bash
$ lua logs
→ All agent logs
→ Look for ❌ ERROR entries
→ Review error messages and stack traces
→ Fix issues in your code
```

### Monitor Performance

```bash
$ lua logs
→ All agent logs  
→ Find ✅ COMPLETE entries
→ Check duration metrics
→ Identify slow operations
```

### Verify Tool Execution

```bash
$ lua logs
→ Specific skill logs
→ Select your skill
→ Look for 🔍 DEBUG entries
→ Verify tool inputs/outputs
```

### Track API Calls

```bash
$ lua logs
→ All agent logs
→ Review API interaction logs
→ Check response times
→ Debug external integrations
```

## Log Entry Details

Each log shows:
- **Timestamp** - When the log was created
- **Type** - Error, debug, info, warn, metric
- **Skill** - Which skill generated the log
- **Tool** - Which tool was executing
- **Message** - Log content
- **Duration** - Time taken (for metrics)
- **Metadata** - User ID, agent ID, etc.

## Navigation Options

<Tabs>
  <Tab title="Next Page →">
    View the next page of logs
    
    Disabled if on last page
  </Tab>

  <Tab title="← Previous Page">
    Go back to previous page
    
    Disabled if on first page
  </Tab>

  <Tab title="🔢 Go to specific page">
    ```
    ? Enter page number (1-88): 45
    ```
    
    Jump directly to any page
  </Tab>

  <Tab title="🔄 Refresh">
    Reload current page with latest logs
    
    Useful for monitoring in real-time
  </Tab>

  <Tab title="❌ Exit">
    Close the logs viewer
  </Tab>
</Tabs>

## Best Practices

<AccordionGroup>
  <Accordion title="Start with All Logs">
    ```bash
    # Get overview first
    lua logs → All agent logs
    
    # Then filter if needed
    lua logs → Specific skill logs
    ```
  </Accordion>

  <Accordion title="Monitor After Deployment">
    ```bash
    # After deploying
    lua push && lua deploy
    
    # Monitor logs
    lua logs → All agent logs → Refresh periodically
    ```
  </Accordion>

  <Accordion title="Debug Specific Issues">
    ```bash
    # Issue with specific skill
    lua logs → Specific skill logs → Select problem skill
    
    # Focus on errors
    # Look for ❌ ERROR entries
    ```
  </Accordion>

  <Accordion title="Track Performance">
    ```bash
    # Check operation speed
    lua logs → All agent logs
    
    # Note duration on ✅ COMPLETE entries
    # Identify slow operations (>1000ms)
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="No logs available">
    **Causes:**
    - Skills not deployed yet
    - Agent hasn't been used
    - Viewing wrong agent
    
    **Solution:**
    ```bash
    lua push  # Deploy skills
    lua chat  # Use agent to generate logs
    lua logs  # View logs
    ```
  </Accordion>

  <Accordion title="Can't find specific error">
    **Try:**
    - Navigate through pages
    - Filter by specific skill
    - Look at recent pages (page 1)
    - Check timestamp for when error occurred
  </Accordion>

  <Accordion title="Logs not updating">
    **Use refresh:**
    ```
    ? Navigation: 🔄 Refresh
    ```
    
    Reloads current page with latest data
  </Accordion>
</AccordionGroup>

## Integration with Workflow

### During Development

```bash
# Make changes
vim src/tools/MyTool.ts

# Test
lua chat

# Check logs immediately
lua logs
# Look for errors or warnings
```

### After Deployment

```bash
# Deploy
lua push && lua deploy

# Monitor logs
lua logs
# Watch for errors in production
# Check performance metrics
```

### Debugging Issues

```bash
# User reports issue
lua logs
# Find relevant error
# Note: timestamp, skill, tool
# Fix in code
# Deploy fix
# Monitor logs again
```
