## Customization Strategies

### Strategy 1: Keep Only What You Need

**Remove unused tools:**

```bash
# Delete files you don't need
rm src/tools/ProductsTool.ts
rm src/tools/BasketTool.ts
rm src/tools/OrderTool.ts

# Update src/index.ts
# Remove imports and tools
```

**Update `src/index.ts`:**

```typescript
import { LuaSkill } from "lua-cli";
import GetWeatherTool from "./tools/GetWeatherTool";

const weatherSkill = new LuaSkill({
  name: "weather-skill",
  version: "1.0.0",
  description: "Provides weather information",
  context: "Use get_weather when users ask about weather",
  tools: [new GetWeatherTool()]
});
```

### Strategy 2: Modify Examples

**Adapt tools to your domain:**

For example, change the movies database to articles:

```typescript
// Rename CustomDataTool.ts → ArticleTool.ts
// Change collection name: 'movies' → 'articles'
// Update fields: title, director → title, content, category

export class CreateArticleTool implements LuaTool {
  name = "create_article";
  description = "Add article to knowledge base";
  
  inputSchema = z.object({
    title: z.string(),
    content: z.string(),
    category: z.string()
  });

  async execute(input: any) {
    const searchText = `${input.title} ${input.content}`;
    return await Data.create('articles', input, searchText);
  }
}
```

### Strategy 3: Mix and Match

**Keep useful examples, add your custom tools:**

```typescript
// Keep user data tools (always useful)
import { GetUserDataTool } from "./tools/UserDataTool";

// Keep custom data tools (for knowledge base)
import { CreateArticleTool, SearchArticlesTool } from "./tools/ArticleTool";

// Add your custom tools
import MyCustomTool from "./tools/MyCustomTool";

const mySkill = new LuaSkill({
  tools: [
    new GetUserDataTool(),
    new CreateArticleTool(),
    new SearchArticlesTool(),
    new MyCustomTool()
  ]
});
```

## Key Files Explained

### src/index.ts

**This is your starting point.** Defines all skills and their tools:

```typescript
import { LuaSkill } from "lua-cli";
import GetWeatherTool from "./tools/GetWeatherTool";

// Define your skills here
const mySkill = new LuaSkill({
  name: "my-skill",
  description: "What it does",
  context: "When to use it",
  tools: [new GetWeatherTool()]
});
```

### src/tools/*.ts

Tool implementations. Each file contains one or more tools:

```typescript
import { LuaTool } from "lua-cli";
import { z } from "zod";

export default class MyTool implements LuaTool {
  name = "my_tool";
  description = "Clear description";
  inputSchema = z.object({ /* ... */ });
  async execute(input) { /* ... */ }
}
```

### lua.skill.yaml

Auto-managed configuration. The CLI updates this automatically:

```yaml
agent:
  agentId: agent_abc123
  orgId: org_xyz789

skills:
  - name: my-skill
    version: 1.0.0
    skillId: skill_xyz  # Auto-created
```

<Warning>
Don't manually edit `skillId` - it's auto-managed by the CLI!
</Warning>
